Bootstrap Fugue icons set

Have you ever whated to use Fugue icons seamlessly. Just like you use Bootstrap icons but in a way it does not interfier with other icons you use like Boostrap glyphicons or FontAwesome? Whithout even worring to copy it to your project and with access to all latest icons?

Here it is. Bootstrap fugue icons. Just link CSS file and that is it.

View on GitHub

Installation

Fugue icons are distributed under CC license and require attribution. If you do not want to provide attribution please purchase royalty free license.

You can install it locally include to your project with bower package manager

$ bower install bootstrap-fugue-icons

Or you can use temporary CDN by including this link right after you include bootstrap.css

<link rel="stylesheet" href="http://bootstrap-fugue.azurewebsites.net/css/bootstrap-fugue-min.css"/>

If we reach high traffic then we will find good CDN service who would sponsor to host our project.

Features

  • Since we have special prefix f- it does not controdict with any other icons or override any icon be it Bootstrap icons or Font Awesome. It means that if you connect this CSS all your old icons will continue to be displayed.

  • You can link 2 CSS files: bootstrap-fugue-min.css or bootstrap-fugue-shadow-min.css to render icons with shadow.

  • Special gray class will create special grayscale effect with normal icon on :hover. This uses pure CSS and works in Firefox 3.5+, IE6-9+, Google Chrome & Safari 6+

    <i class="icon-f-abacus gray"></i>

    Here is the demo

    It will aslo work inside a button Timer

Why not sprite technique?

First of all sprite technique is good if you have no too many icons. Estimation sprite size for whole Fugue set is around 2Mb. And I am sure no one will use even 5% of those icons in single project. So it is absolutely useless to load all icons in order to show only few of them. When you have sprite it alway loads escessive amount of data on first load. Icons may viray from page to page so acualy half of the sprite should not be acually loaded even if your sprite contain only icons you use through your project.

If you use something like FontAwesome you get the same problem. In order to render even one icon whole font library have to be loaded.

This technique when we load single icon file on every class does not delay page render. Page is shown immediately and the start to render icon by icon. And all icons are cached along with CSS which is quite big :)
